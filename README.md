CS210 Module Eight Journal - Corner Grocer Project
Project Summary
For this course, I selected my Corner Grocer Item‑Tracking Program as the artifact for my portfolio. This project solves the problem of analyzing daily purchase records by reading an input file, counting how often each item appears, and presenting the results through a menu‑driven interface. The program also generates a backup data file automatically. This helps the Corner Grocer understand customer buying patterns and reorganize their store layout based on real data.
What I Did Well
I implemented a clean and organized class structure using a GroceryTracker class that encapsulates all frequency‑tracking logic. I also used a map<string, int> to efficiently store and retrieve item frequencies. My code is readable, well‑commented, and follows consistent naming conventions. The menu system is simple and intuitive, and the histogram output is formatted clearly.
Where I Could Enhance My Code
One improvement would be adding more robust input validation and error handling. For example, I could sanitize user input to handle case sensitivity or unexpected characters. I could also expand the program to support larger datasets by reading line‑based entries instead of single words. These enhancements would make the program more secure, more flexible, and more user-friendly.
Most Challenging Parts and How I Overcame Them
The most challenging part was designing the class so that all file processing and frequency logic stayed organized and reusable. I overcame this by breaking the program into small, focused functions and testing each one individually. I also relied on zyBooks examples, Visual Studio debugging tools, and online C++ references to reinforce my understanding of maps, file I/O, and exception handling.
Skills That Transfer to Other Projects
This project strengthened several transferable skills:
• Object-oriented design using classes
• File input/output for reading and writing data
• Use of STL containers, especially maps
• Menu-driven program design
• Debugging and testing in Visual Studio
These skills apply directly to future programming courses and real‑world software development.
How I Made the Program Maintainable, Readable, and Adaptable
I used clear naming conventions, consistent indentation, and in‑line comments to explain key logic. The class‑based structure makes the program easy to extend - for example, adding new menu options or supporting different file formats. Separating data processing from user interaction also improves maintainability and readability.
